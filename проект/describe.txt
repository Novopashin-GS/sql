База данных предназначена для производства. 
Имеются следующие таблицы - сотрудники, отделы, должности, права доступа сотрудников, сообщения, оборудованя, отчеты о проделанной работе, журнал отметок времени, склад, журнал о взятии ресурсов со склада.
С помощью этих данных можно вести подсчет компонентов на складе, смотреть отчеты сотрудников, смотреть отметки времени на обеды, приходы на работу, уходы на работу и прочее.
В файле create_database_tables создание базы данных, таблиц.
В файле create_data тестовые данные таблиц.
В файле скрипты характерных выборок:
update заменяет получателя сообщений, чтобы каждый не писал сам себе;
первый select считает колличество оборудования у отделов;
второй select считает колличество отчетов по работе каждого сотрудника;
третий select считает сколько раз каждый сотрудник ходил на обед за месяц;
первый join находит имя и фамилию сотрудника, а также что он брал со склада;
второй jpin находит оборудования, которые находятся в зоне ответственности каждой должности.
В файле views:
первое представление убирает лишние данные для, например, администраторов;
второе представление лишь проверяет наличие отчетов по работе.
В файле trigger_procedure:
триггер автоматически уменьшает колличество компонентов на складе, после того как компонент взяли;
процедура сразу выводит компоненты, которые закончились.
